# XiaoTian v0.2.2 更新日志

> 发布日期: 2025-03-15

## 核心工作

本版本专注于代码重构和架构优化，将命令解析和执行逻辑分离，并改进了更新去重机制，使系统更加模块化和可扩展。

### 命令系统重构

- 实现了全新的命令模块：
  - 创建 `Command` 枚举，用于表示所有支持的命令
  - 实现 `CommandParser` 负责命令解析与验证
  - 为每种命令类型创建专用的子枚举（`AddCommand`, `ListCommand` 等）
- 重构处理模块：
  - 设计并实现 `CommandProcessor` trait，负责命令执行
  - 创建专门的处理程序类（`AddHandler`, `ListHandler` 等）
  - 采用工厂模式来创建和管理处理程序
- 优化了 REPL 实现：
  - 使用新的命令系统替换了原有的命令处理逻辑
  - 改进错误处理和用户反馈

### 配置管理简化

- 移除了配置相关命令（`config get/set/list`）
- 简化了用户界面，减少命令数量
- 配置仍通过配置文件管理，但不再通过命令行修改

### 更新去重机制

- 实现智能去重机制，避免 `fetch updates` 重复执行导致的数据重复
- 设计分层去重策略：
  - 为不同更新类型创建专用的去重辅助函数
  - 实现基于特定标识符的精确匹配（如 commit SHA, issue/PR number）
  - 当无法获取特定标识符时，回退到基于标题和日期的比较
- 重构存储层的 `save_update` 方法，集成去重逻辑

### 代码质量改进

- 提高了代码的模块化程度和可测试性
- 优化了错误处理机制，提供了更准确的错误信息
- 添加了完整的代码文档和注释
- 统一了代码风格，所有注释均使用英文
- 为关键功能添加了单元测试

## 技术栈更新

- 采用了更现代的 Rust 设计模式
- 引入了 `async-trait` 支持异步特性实现
- 使用 `thiserror` 改进错误处理
- 使用 `DashMap` 进行并发安全的内存存储

## 已知问题

- 配置更改需要重启应用才能生效
- 存储仍仅限于内存中，重启后数据会丢失
- 缺少自动化测试以验证去重逻辑的有效性
- 未实现对非 GitHub 源的支持

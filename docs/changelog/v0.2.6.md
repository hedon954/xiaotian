# XiaoTian v0.2.6 更新日志

> 发布日期: 2025-03-11

## 核心工作

本版本实施了重构计划的第一阶段，优化了 ID 系统，使其更简单、直观，同时为后续重构奠定基础。

### ID 系统简化

- **采用整数 ID 替代 UUID**:

  - 将 Repository 和 Subscription 的 ID 从 UUID 改为自增整数 (i32)
  - 在 MemoryStorage 中添加自增 ID 计数器，确保唯一性
  - 保留 Update 的 UUID ID，以应对高频率、并发的更新场景

- **模型关系优化**:

  - 订阅直接使用整数 ID 引用其源
  - 更新使用整数 ID 引用源，保持数据完整性
  - 优化了存储层接口，使其更加一致和直观

- **用户体验改进**:

  - 命令行交互使用简单的整数 ID，如 `show repository 1` 和 `delete repository 1`
  - 添加 `subscribe <repository_id>` 命令，使得订阅操作更加直观
  - 错误信息更加明确，包含具体的实体类型和 ID
  - 简化了 ID 的输入，提高了命令行操作效率

- **错误处理增强**:
  - 重新设计 StorageError，包含更多上下文信息
  - 添加 InvalidIdentifier 错误类型，提供更精确的错误反馈
  - 统一了返回值设计，如删除操作返回布尔值表示成功与否

## 技术优化

- **存储接口改进**:

  - `get_repository` 方法现在返回 `Option<Repository>`，与 Rust 惯例保持一致
  - 所有接口保持一致使用整数 ID，减少类型转换
  - 删除操作返回更明确的结果，便于错误处理

- **代码一致性提升**:
  - 统一了 ID 处理方式，减少混用不同类型 ID 的问题
  - 存储方法签名更加一致，易于理解和维护
  - 构造函数接受 ID 参数，强化类型安全

## 已知问题与限制

- 当前版本仅完成 ID 系统优化，尚未实现存储层持久化
- 应用重启后内存中数据会丢失，ID 计数器将重置
- Update 的 ID 仍使用 UUID，与其他实体类型不同

## 后续计划

本版本是重构计划的第一步，后续版本将继续优化：

1. 实现模型解耦，将 Subscription 与特定源类型进一步分离
2. 简化整体架构，优化命令处理流程
3. 实现持久化存储，确保数据在应用重启后能够恢复

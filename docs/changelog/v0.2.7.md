# XiaoTian v0.2.7 更新日志

> 发布日期: 2025-03-15

## 核心工作

本版本实施了重构计划的第二阶段，大幅简化了应用架构，专注于核心功能，优化了用户体验，显著降低了系统复杂度。

### 架构简化

- **移除 Subscription 概念**:

  - 取消 Source 和 Subscription 的分离，直接使用 Source 进行管理
  - 为每个 Source 添加自定义的 `fetch_update` 函数，实现特定源类型的更新获取逻辑
  - 简化数据模型关系，减少实体间的复杂依赖
  - 移除了相关的 Subscription 命令，简化用户操作流程

- **取消 Update 存储**:

  - 不再持久化存储 Update 数据，转为即时处理模式
  - 每次执行 `fetch_update` 时直接输出结果，而非存储后再查询
  - 为后续的定时任务和报告生成功能做好准备
  - 大幅降低了存储需求和系统复杂度

- **命令简化**:

  - 将 `repository` 简化为 `repo`，`repositories` 简化为 `repos`
  - 移除了与 Subscription 和 Update 相关的命令
  - 保留核心功能命令，减少用户学习成本
  - 命令名称更短，提高使用效率

- **界面体验优化**:
  - 简化了命令参数和选项，使命令更加直观
  - 优化了输出格式，使结果更易于阅读
  - 统一了命令风格和反馈方式
  - 错误提示更加清晰和针对性

## 技术优化

- **数据模型精简**:

  - 移除了 Subscription 实体类型及相关代码
  - 简化了 Source 实体，增加了特定功能方法
  - 不再使用 Update 实体进行存储
  - 优化了实体间的关系，减少了代码复杂度

- **命令处理改进**:

  - 简化了命令处理流程
  - 更新了命令解析逻辑，适应新的命名约定
  - 优化了命令处理器的结构
  - 改进了命令执行的反馈机制

- **函数接口优化**:
  - 为 Source 添加了 `fetch_update` 函数接口
  - 统一了各类源的更新获取逻辑
  - 简化了命令实现，减少重复代码
  - 提高了系统扩展性，便于添加新的源类型

## 已知问题与限制

- 当前版本尚未实现定时触发 `fetch_update` 的功能
- 更新结果直接输出，不提供历史查询能力
- 移除 Update 存储后，无法进行更新去重和差异对比
- 简化的架构可能在支持多种源类型时需要进一步优化

## 后续计划

本版本完成了架构的简化和基础功能调整，后续版本将重点实现：

1. 开发增量更新检测机制
2. 实现定时执行 `fetch_update` 的功能
3. 添加更新报告生成功能，支持多种格式输出
4. 扩展源类型支持，添加不同类型源的 `fetch_update` 实现

# XiaoTian v0.3.0 变更记录

## 版本概述

v0.3.0 实现了定时更新任务系统和报告生成功能，增强了系统的自动化能力，使其能够定期检查仓库更新并生成结构化报告。

## 主要更新

### 1. 定时任务系统

- 使用 `cron_tab` 库实现了基于 cron 表达式的任务调度
- 创建了独立的二进制入口点 `cron` 用于运行定时任务
- 支持自定义 cron 表达式配置更新频率，默认每小时执行一次
- 实现了并发更新处理，提高执行效率

### 2. 报告生成功能

- 设计并实现了结构化的 Markdown 报告模板
- 按源类型和仓库组织报告文件
- 实现了 `reports/{source_type}/{owner}_{repo}_{since}_{until}.md` 的目录结构
- 报告包含完整的更新信息，包括 Pull Requests、Issues 和 Releases
- 优化了报告内容，专注于高价值信息（移除了 Commits 详情，因为这些信息已在 PR 和 Releases 中概括）

### 3. 代码架构优化

- 重构了任务调度逻辑，将复杂业务逻辑移至 `process/schedule.rs`
- 实现了 `ScheduleHandler` 用于管理定时任务
- 改进了错误处理和日志记录
- 增强了模块化设计，提高了代码可维护性

## 技术细节

- 使用 `cron_tab::AsyncCron` 实现异步定时任务
- 支持 UTC+8 (中国标准时间) 的任务调度
- 实现了优雅的错误处理，确保单个源失败不影响其他源
- 添加了自动初始化和示例仓库配置

## 命令行界面

```
# 启动定时任务服务
cargo run --bin cron
```

## 已知问题

- 当前版本的报告只保存在本地，未来版本将考虑添加远程发送功能
- 目前只支持内存存储，未来版本将添加持久化存储支持

## 未来计划

- 添加报告发送功能（如 Email、Webhook）
- 实现任务管理命令（添加、删除、查看任务状态）
- 支持更多源类型的定时更新

# XiaoTian v0.7.0 变更日志

## 主要特性

### 代码架构优化

v0.7.0 版本对 XiaoTian 的代码架构进行了重大优化，大幅简化了系统结构，使其更加专注于核心功能。

- **精简代码架构**：

  - 移除了复杂的订阅系统，改为直接管理仓库源
  - 优化了内部数据流，减少了冗余处理
  - 重构处理器模块，使其更加精简高效
  - 重命名 scheduler 为 reporter，更准确地反映其功能

- **Python 绑定实现**：

  - 使用 PyO3 框架实现 Rust 到 Python 的 FFI 绑定
  - 提供完整的核心功能访问接口
  - 添加了与 Python 环境的数据类型转换
  - 在 `xiaotian-py-binding` 目录中实现了绑定代码

- **Gradio 前端界面**：
  - 基于 Gradio 实现了直观的 Web 用户界面
  - 支持源代码选择和模型选择
  - 提供摘要生成和报告显示功能
  - 支持通过邮件发送生成的报告
  - 在 `frontend-gradio` 目录实现完整前端功能

## 技术改进

### Python 绑定系统

- **FFI 接口设计**：

  - 实现了 `PyProcessor` 类，封装核心 Rust 功能
  - 提供 `get_source_list` 接口获取可用源
  - 提供 `get_model_list` 接口获取可用模型
  - 实现 `fetch_updates` 接口生成更新摘要报告

- **底层集成**：
  - 优化了 Rust 与 Python 之间的数据转换
  - 添加了线程安全和异常处理机制
  - 集成 Tokio runtime 管理异步操作

### 前端与后端集成

- **Gradio UI 设计**：

  - 设计简洁直观的用户界面
  - 实现源仓库选择组件
  - 实现 LLM 模型选择功能
  - 添加邮箱输入和发送控制
  - 提供报告下载和显示功能

- **后端接口**：
  - 开发 `xiaotian_api.py` 与 Rust 后端通信
  - 实现数据格式化和传输功能
  - 添加错误处理和回退机制

### 代码优化

- **翻译与国际化**：

  - 将代码注释和文档从中文翻译为英文
  - 保持错误消息和用户界面的双语支持

- **LLM 集成改进**：

  - 更新 Ollama 配置，支持最新的 Llama3 模型
  - 优化提示词模板，提升生成报告质量
  - 添加温度和 top_p 参数控制

- **通知系统完善**：
  - 改进邮件通知系统
  - 优化通知消息格式和样式
  - 添加发送后的确认和错误处理

## 使用示例

### 通过 Web 界面使用

1. 启动 Gradio Web 界面：

```bash
cd frontend-gradio
python run.py
```

2. 在浏览器中访问 `http://localhost:7860`
3. 选择 GitHub 源和 LLM 模型
4. 输入邮箱接收通知（可选）
5. 点击生成按钮开始分析

### 通过命令行使用

```bash
# 构建项目
cargo build --release

# 启动定时更新任务
cargo run --bin cron
```

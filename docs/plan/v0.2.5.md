# XiaoTian v0.2.5 数据关系优化计划

## 数据关系完整性与级联操作

| 任务               | 描述                                             | 优先级 |
| ------------------ | ------------------------------------------------ | ------ |
| 引用完整性检查     | 为 Subscription 与 Source 之间添加引用完整性约束 | 高     |
| 级联删除实现       | 实现 Source 删除时的级联删除机制                 | 高     |
| 交互式确认流程     | 添加删除前的确认机制和关联项目提示               | 高     |
| 错误处理优化       | 改进与数据关系相关的错误处理和提示               | 中     |
| 数据一致性检查工具 | 开发用于检查数据一致性的工具                     | 中     |
| 关系完整性文档     | 更新文档，说明模型间的引用约束                   | 低     |

## 详细任务说明

### 引用完整性检查

- **添加 Subscription 时的 Source 验证**：

  - 当创建新的 Subscription 时，验证对应的 Source 是否存在
  - 如果 Source 不存在，返回明确的错误信息
  - 在`Storage` trait 中添加检查方法
  - 在`Processor`的`add_subscription`方法中实现验证

- **访问 Subscription 时的关联检查**：
  - 获取 Subscription 详情时，检查对应的 Source 是否仍然存在
  - 如果 Source 已不存在，提供警告信息

### 级联删除实现

- **Source 删除前的关联项检查**：

  - 添加方法检查与特定 Source 关联的所有 Subscription
  - 实现列出关联项的功能

- **确认机制与级联删除**：

  - 在删除 Source 前提示用户确认
  - 显示将被级联删除的所有 Subscription 和 Update 数量
  - 实现级联删除逻辑，确保关联数据一并删除

- **删除后的清理与验证**：
  - 验证所有关联数据是否已成功删除
  - 优化存储层以支持高效的级联删除

### 错误处理优化

- **数据关系错误类型**：

  - 添加新的错误类型以表示数据关系问题
  - 例如：`ReferenceError`、`OrphanedSubscriptionError`等

- **具体错误信息**：
  - 提供更加具体和有帮助的错误信息
  - 在适当情况下提供修复建议

### 用户体验优化

- **交互式确认流程**：

  - 实现多步骤确认过程，特别是对于潜在危险的操作
  - 使用颜色和格式强调重要信息

- **关联项目可视化**：
  - 提供简洁的方式显示关联项目
  - 例如，在删除 Source 时列出受影响的 Subscription 名称

## 预期成果

- 更强健的数据模型关系
- 更安全的删除操作，防止孤立数据
- 改进的用户体验，特别是对于可能有副作用的操作
- 更清晰的错误信息和提示
- 减少因数据不一致导致的问题
